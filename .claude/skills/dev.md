# /dev 워크플로우

사용자가 `$ARGUMENTS`를 요청했습니다. 아래 3단계 워크플로우를 순서대로 실행하세요.

## 단계 1: 요구사항 분석 및 계획 (requirement-planner)

Task 도구로 `requirement-planner` 서브에이전트를 실행하세요.

프롬프트:
```
사용자 요청: "$ARGUMENTS"

이 요청을 분석하고 구체적인 구현 계획을 수립해주세요.
- 코드베이스를 먼저 탐색해서 기존 패턴을 파악하세요
- 영향받는 파일과 구체적인 작업 목록을 정리하세요
- 계획을 한국어로 작성하세요
```

requirement-planner의 결과를 사용자에게 보여주고 **승인을 받으세요**:
- "이 계획대로 진행할까요?" 라고 물어보세요
- 사용자가 수정을 요청하면 반영하세요
- 사용자가 승인하면 다음 단계로 진행하세요

## 단계 2: 코드 구현 (senior-code-architect)

사용자 승인 후, Task 도구로 `senior-code-architect` 서브에이전트를 실행하세요.

프롬프트:
```
사용자 요청: "$ARGUMENTS"

아래 구현 계획에 따라 코드를 작성하세요:

[1단계에서 받은 계획 내용을 여기에 포함]

- 기존 코드베이스의 패턴을 따르세요
- 비전공자도 이해할 수 있도록 한국어 주석을 달아주세요
- 클린 아키텍처 원칙을 지키세요
```

구현이 완료되면 변경 사항 요약을 사용자에게 보여주세요.

## 단계 3: 품질 검증 (quality-guard)

코드 구현 완료 후, Task 도구로 `quality-guard` 서브에이전트를 실행하세요.

프롬프트:
```
방금 다음 작업의 코드가 구현되었습니다: "$ARGUMENTS"

ESLint, TypeScript 타입 체크, 빌드 검증을 모두 실행하고 결과를 보고해주세요.
```

### 품질 검증 결과 처리:
- **모두 통과**: 사용자에게 완료 보고
- **오류 발견**: 오류 내용을 확인하고 `senior-code-architect`를 다시 실행하여 수정 → 다시 `quality-guard` 실행 (최대 2회 반복)

## 최종 보고

모든 단계가 완료되면 다음을 사용자에게 보여주세요:

```
## 개발 완료 보고

### 요청 사항
$ARGUMENTS

### 변경된 파일
- [파일 목록]

### 품질 검증
- ESLint: ✅/❌
- TypeScript: ✅/❌
- Build: ✅/❌

### 다음 단계 (있는 경우)
- [추가 작업이 필요한 경우 안내]
```
